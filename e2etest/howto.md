# 正常系E2E 手順書

## スクリーンショット運用ルール

画面操作においては、**できるだけこまめにスクリーンショットを取得**してください。

* **各操作ステップの前後**でスクリーンショットを撮り、意図した状態になっているかを確認する
* **座標指定によるタップ・クリック操作**は特に注意が必要です。座標はデバイスや解像度によってずれやすいため、以下を徹底してください：

  1. 操作前にスクリーンショットを取得し、対象要素の座標を確認する
  2. 操作後に必ずスクリーンショットを取得し、意図した要素が操作されたかを確認する
  3. 想定と異なる画面状態になっていた場合は、再度スクリーンショットで現在地を確認してから次の操作に進む

* **画面遷移のたびに**スクリーンショットを取得し、正しい画面に遷移したことを確認する
* スクリーンショットの結果をもとに、次の操作の座標・要素を都度判断すること（キャッシュした座標をそのまま使い回さない）

## 0. 前提・テストデータ

* テストユーザー：

  * Username：`demo_user`
  * Password：`password`
* Seedデータ：

  * 利用明細が10件以上存在する
  * 一覧は **「日付：新しい順（降順）」** が正しい期待値
* ディープリンク：

  * スキーム：`e2etestflutter://`
  * テスト用URL：`e2etestflutter:///transactions/tx_0006`（Amazon の明細）
  * adbコマンド：`adb shell am start -a android.intent.action.VIEW -d “e2etestflutter:///transactions/tx_0006”`
* 明細ID一覧（Seedデータ）：

  * `tx_0000` AWS, `tx_0001` Netflix, `tx_0002` Uber Eats, `tx_0003` JR East, `tx_0004` Amazon
  * `tx_0005` Spotify, `tx_0006` Amazon, `tx_0007` Starbucks, `tx_0008` Uniqlo, `tx_0009` GitHub, `tx_0010` Lawson

## 1. ディープリンク → ログイン → 明細確認

### 操作

1. アプリが起動していない状態で、ディープリンクを実行する

   * `adb shell am start -a android.intent.action.VIEW -d “e2etestflutter:///transactions/tx_0006”`
2. ログイン画面が表示されるので以下を入力

   * Username：`demo_user`
   * Password：`password`
3. 「Login」ボタンを押下

### 期待結果

* ログイン後、**ホーム画面ではなく明細詳細画面**に直接遷移する
* 明細詳細画面に以下が表示される

  * merchant：Amazon
  * amount：金額が表示される（空でない）
  * timestamp：日時が表示される
  * status：ステータスが表示される
  * 取引ID：`tx_0006`
* 画面下部に**ボトムタブバー**（ホーム・取引・設定）が表示される
* 「取引一覧を見る」ボタンが表示される

---

## 2. 明細詳細 → 取引一覧へ遷移

### 操作

1. 明細詳細画面で「取引一覧を見る」ボタンをタップ（または画面下部の「取引」タブをタップ）

### 期待結果

* 利用明細一覧が表示される
* 件数が表示される（10件以上が望ましい）
* ソートラベルが **「日付：新しい順」** と表示される
* 一覧の各行に以下が表示される

  * merchant（加盟店名）
  * amount（金額）
  * timestampISO（ISO文字列）
  * status（Posted/Pending）
* ボトムタブバーが引き続き表示されている

---

## 3. “日付：新しい順（降順）”の検証（正常の肝）

ここがデモの基礎体力になります。E2Eでも人手でも、最低限これを確認できればOK。

### 操作

1. 一覧の **先頭行（1件目）** の `timestampISO` を記録
2. 一覧の **2行目（2件目）** の `timestampISO` を記録

### 期待結果

* 1件目のtimestampが、2件目より **新しい（降順）**

  * 例：

    * 1件目：`2026-01-29T10:00:00Z`
    * 2件目：`2026-01-18T10:30:00Z`
  * ISO文字列として比較しても良いし、表示された日付で判断しても良い

---

## 4. 明細詳細の表示整合（一覧→詳細）

### 操作

1. 一覧の先頭行をタップして明細詳細へ遷移
2. 詳細画面で以下を確認

   * merchant
   * amount
   * timestamp（ISO＋ローカル表示でも可）
   * status

### 期待結果

* 詳細画面の merchant/amount/timestamp/status が、一覧でタップした行の値と一致する
* 画面がクラッシュしない、空欄にならない

---

## 5. ボトムタブでホーム画面へ遷移

### 操作

1. 画面下部の「ホーム」タブをタップ

### 期待結果

* ホーム画面へ遷移する
* ユーザー名表示が `demo_user` である
* 今月の支出合計（monthlyTotal）が表示される（値は固定でOK、非空なら可）
* 「取引を確認する」「設定」への導線ボタンが表示される
* ボトムタブバーが表示され、「ホーム」タブが選択状態である

---

## 6. 設定変更 → 永続化確認（状態保持）

### 操作

1. 画面下部の「設定」タブをタップして設定画面へ移動
2. 次のトグルを順に操作（例）

   * 通知：OFF→ON（またはON→OFF）
   * 金額マスク：OFF→ON
3. 「ホーム」タブをタップしてホームへ戻る
4. **ログアウト**（設定タブ → ログアウトボタン）
5. 再度ログイン（手順1と同じユーザー情報）

### 期待結果

* 再ログイン後も、設定変更が保持されている

  * 通知トグルが変更後の状態のまま
  * 金額マスクが変更後の状態のまま
* 金額マスクがONの場合、ホーム画面および利用明細一覧で金額が「****」でマスクされて表示される
* 金額マスクがOFFの場合、金額が通常通り数値で表示される

# 手順の「合格/不合格」判定基準（簡潔版）

* 合格：

  * ディープリンクからログイン後、指定した明細詳細に直接遷移する
  * ボトムタブでホーム・取引・設定を自由に行き来できる
  * 明細一覧が表示され、先頭2件が降順
  * 一覧→詳細で値が一致
  * 設定が保存され、再ログイン後も保持
* 不合格：

  * ディープリンクでログイン後にホームに飛ばされる（明細詳細に遷移しない）
  * ボトムタブが表示されない、タブで画面遷移できない
  * 画面遷移できない、クラッシュ、一覧が空、ソートが降順になっていない、設定が保持されない
